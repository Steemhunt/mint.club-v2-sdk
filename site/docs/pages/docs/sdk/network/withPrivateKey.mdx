# withPrivateKey

`withPrivateKey` configures the selected network client with a private-key account.

## Usage

:::warning[WARNING]
Use this only in a trusted server process. Never expose a private key in browser code, logs, prompts, command arguments, or source control.
:::

```ts [example.ts]
import { mintclub } from '@mint.club/v2-sdk'

await mintclub
  .network('ethereum')
  .withPrivateKey(process.env.PRIVATE_KEY as `0x${string}`) // [!code focus]
  .token('MINT')
  .create({
    name: 'Mint Club',
    reserveToken: {
      address: '0xC02aaA39b223FE8D0A0e5C4F27eAD9083C756Cc2', // mainnet WETH token address
      decimals: 18,
    },
    curveData: {
      curveType: 'LINEAR',
      stepCount: 10,
      maxSupply: 10_000,
      initialMintingPrice: 0.01, // 0.01 WETH
      finalMintingPrice: 0.1, // 0.1 WETH
      creatorAllocation: 100,
    },
  })
```

## Return Value

Returns the network-scoped client, so you can continue with [`token()`](/docs/sdk/network/token), [`nft()`](/docs/sdk/network/nft), or another network helper.

## Parameters

### privateKey

- Type: `'0x${string}'`

The private key of a wallet.
