# create [Deploys a Bonding Curved ERC-1155 Contract]

## Usage

Below is a basic example of how to call the `create` function.

:::tip[NOTE]
When using the `create` function, you must pass a `symbol` that you want to create as the parameter of the `nft` function.
```ts
await mintclub
  .network('ethereum')
  .nft('MNFT') // [!code focus]
```

> In the example below, we are creating a nft with the symbol `MNFT` with the nft name of `Mint Club Collection` on the `ethereum` network.
::::



:::code-group

```ts [curvedata-example.ts]
import { curveData } from './data';

await mintclub
  .network('ethereum')
  .nft('MNFT')
  .create({
    name: 'Mint Club Collection',
    reserveToken: {
      address: '0xC02aaA39b223FE8D0A0e5C4F27eAD9083C756Cc2', // mainnet WETH nft address 
      decimals: 18, 
    }, 
    image:  'ipfs://...' | 'https://...' | File,
	curveData,
  })
```

```ts [stepdata-example.ts]
import { stepData } from './data';

await mintclub
  .network('ethereum')
  .nft('MNFT')
  .create({
    name: 'Mint Club Collection',
    reserveToken: {
      address: '0xC02aaA39b223FE8D0A0e5C4F27eAD9083C756Cc2', // mainnet WETH nft address 
      decimals: 18, 
    }, 
    image:  'ipfs://...' | 'https://...' | File,
	stepData, 
  })
```

```ts [data.ts]
// Only one of curveData or stepData is required

export const stepData = [
	{ rangeTo: 50, price: 0 }, // Since price is 0, the first 50 tokens will be allocated to the creator
	{ rangeTo: 100, price: 0.01 }, // 0.01 WETH for the next 50 tokens
	{ rangeTo: 1000, price: 0.1 }, // 0.1 WETH for the next 900 tokens
	{ rangeTo: 10_000, price: 1 }, // 1 WETH for the next 9000 tokens
]

export const curveData = {
	curveType: 'LINEAR', 
	stepCount: 10,
	maxSupply: 10_000,
	initialMintingPrice: 0.01, // 0.01 WETH 
	finalMintingPrice: 0.1, // 0.1 WETH
	creatorAllocation: 100,
}
```

:::

:::warning[NOTE]
If you pass a `File` instance, you will also need to pass a [`filebaseApiKey`](#filebaseapikey-conditionally-required) as a parameter.
:::

## Return Value

[`TransactionReceipt`](/docs/sdk/network/transactions/receipt) | `undefined`

The receipt for the signed transaction.

## Parameters

### name

- Type: `string`

The nft name.

```ts
await mintclub
  .network('ethereum')
  .nft('MNFT')
  .create({
    name: 'Mint Club Collection', // [!code focus]
    reserveToken: {
      address: '0xC02aaA39b223FE8D0A0e5C4F27eAD9083C756Cc2', // mainnet WETH nft address
      decimals: 18,
    },
    image,
    curveData,
  })
```

### reserveToken

- Type: `{ address: '0x${string}'; decimals: number }`

The reserve nft to use for the bonding curve. This is the nft that will be used to mint new tokens.

```ts
await mintclub
  .network('ethereum')
  .nft('MNFT')
  .create({
    name: 'Mint Club Collection',
    // [!code focus]
    reserveToken: { // [!code focus]
      // [!code focus]
      address: '0xC02aaA39b223FE8D0A0e5C4F27eAD9083C756Cc2', // mainnet WETH nft address // [!code focus]
      decimals: 18, // [!code focus]
    }, // [!code focus]
    image,
    curveData,
  })
```

### image

- Type: `File` | `'ipfs://${string}'` | `'https://${string}'`

The image to use for the nft. You can pass a `File` instance, an `IPFS Hash`, or a `URL`.


:::warning[NOTE]
If you pass a `File` instance, you will also need to pass a [`filebaseApiKey`](#filebaseapikey-conditionally-required) as a parameter.
:::

:::code-group

```ts [file-example.ts]
await mintclub
  .network('ethereum')
  .nft('MNFT')
  .create({
    name: 'Mint Club Collection',
    reserveToken: {

      decimals: 18,
    }, 
    filebaseApiKey: '[Your Filebase API KEY]' // [!code focus]
    image: URL.createObjectURL('...') // [!code focus]
    // or a File instance when using in node.js // [!code focus]
    curveData,
  })
```

```ts [url-example.ts]
await mintclub

    }, 
    image: 'ipfs://...' | 'https://...' // [!code focus]
    curveData,
  })
```

### filebaseApiKey (conditionally required)

- Type: `string`


:::tip[Filebase]
[Filebase](https://filebase.com/) is a free, decentralized storage solution that is used to store the media files of the nft.
:::


```ts

    }, 
    curveData,
    image,
    filebaseApiKey: '[Your Filebase API Key]' // [!code focus]
  })
```



### curveData (conditionally required)

If [`stepData`](#stepdata-conditionally-required) is not provided, `curveData` is required.

It will automatically generate a bonding curve data with the given curve type for you.

#### `curveData` is an object with the following properties:

##### curveType

- Type: `string`

Can be one of : `"LINEAR"` `"EXPONENTIAL"` `"LOGARITHMIC"` `"FLAT"`

##### stepCount

- Type: `number`

This is the number of steps the curve will have. The more steps, the more granular the curve will be. Since this data is stored on-chain, Greater stepCount will result in higher gas costs.

##### maxSupply

- Type: `number`

The maximum supply of the nft.

##### initialMintingPrice

- Type: `number`

The initial price of the nft on the bonding curve.

##### finalMintingPrice

- Type: `number`

The final price of the nft on the bonding curve. 


##### creatorAllocation (optional)

- Type: `number`
- Default: `0`

The amount of tokens that will be allocated to the creator of the bonding curve upon deployment.

:::code-group

```ts [example.ts]
import { curveData } from './curve-data';

await mintclub
  .network('ethereum')
  .nft('MNFT')
  .create({
    name: 'Mint Club Collection',
    reserveToken: {
      address: '0xC02aaA39b223FE8D0A0e5C4F27eAD9083C756Cc2', // mainnet WETH nft address 
      decimals: 18, 
    }, 
    image,
		// [!code focus]
	curveData, // [!code focus]
		// [!code focus]
  })
```

```ts [curve-data.ts]
export const curveData = {
	curveType: 'LINEAR', 
	stepCount: 10,
	maxSupply: 10_000,
	initialMintingPrice: 0.01, // 0.01 WETH 
	finalMintingPrice: 0.1, // 0.1 WETH
	creatorAllocation: 100,
}
```

### stepData (conditionally required)

- Type: `{ rangeTo: number; price: number }[]`

If [`curveData`](#curvedata-conditionally-required) is not provided, `stepData` is required.

Steps that will be used to construct the bonding curve. The `rangeTo` property is the number of tokens that will be minted until the given price. The `price` property is the price of the nft in the reserve nft.

If price is 0, the first `rangeTo` tokens will be allocated to the creator of the bonding curve.

:::warning[NOTE]
> Note that the value of `rangeTo` and `price` can NOT be decreasing.
:::

:::code-group

```ts [example.ts]
import { stepData } from './step-data';

await mintclub
  .network('ethereum')
  .nft('MNFT')
  .create({
    name: 'Mint Club Collection',
    reserveToken: {
      address: '0xC02aaA39b223FE8D0A0e5C4F27eAD9083C756Cc2', // mainnet WETH nft address 
      decimals: 18, 
    }, 
    image,
		// [!code focus]
	stepData, // [!code focus]
		// [!code focus]
  })
```

```ts [step-data.ts]
export const stepData = [
	{ rangeTo: 50, price: 0 }, // Since price is 0, the first 50 tokens will be allocated to the creator
	{ rangeTo: 100, price: 0.01 }, // 0.01 WETH for the next 50 tokens
	{ rangeTo: 1000, price: 0.1 }, // 0.1 WETH for the next 900 tokens
	{ rangeTo: 10_000, price: 1 }, // 1 WETH for the next 9000 tokens
]
```

### buyRoyalty (optional)

- Type: `number`
- Default: `0.03`

buyRoyalty is the percentage of the buy trade fee that will go to the creator.

:::note[NOTE]
The royalty is split 8:2 between the creator (80%) and the protocol (20%) [Link](https://docs.mint.club/royalties/creator-royalties)
:::


```ts
await mintclub
  .network('ethereum')
  .nft('MNFT')
  .create({
    name: 'Mint Club Collection',
    reserveToken: {
      address: '0xC02aaA39b223FE8D0A0e5C4F27eAD9083C756Cc2', // mainnet WETH nft address 
      decimals: 18,
    }, 
    image,
    curveData,
    buyRoyalty: 0.03, // [!code focus]
  })
```

### onIpfsUploadStart (optional)

- Type: `onIpfsUploadStart?: () => void;` 
- Default: `undefined`

Callback function for when the media starts to upload to IPFS.

This callback function will only be invoked when you pass a `File` instance for the `image` or `video` parameter along with the filebaseApiKey.

:::warning[NOTE]
This will not be called when you pass an IPFS Hash or a URL
:::

```ts
await mintclub
  .network('ethereum')
  .nft('MNFT')
  .create({
    name: 'Mint Club Collection',
    reserveToken: {
      address: '0xC02aaA39b223FE8D0A0e5C4F27eAD9083C756Cc2', // mainnet WETH nft address 
      decimals: 18,
    }, 
    image,
    curveData,
	onIpfsUploadStart: () => {} // [!code focus]
  })
```
