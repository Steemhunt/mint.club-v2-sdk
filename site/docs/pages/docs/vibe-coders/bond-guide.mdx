# Bond operations

Use the network-scoped `bond` helper for protocol-level Bond reads and royalty claims. Use `token()` or `nft()` for asset trading and creation.

```ts
import { mintclub } from '@mint.club/v2-sdk'

const bond = mintclub.network('base').bond
```

## Creation fee

```ts
const creationFee = await bond.getCreationFee()
```

The result is the live native-currency fee in its smallest unit. Pass it through unchanged when a lower-level flow requires the creation fee; format it only for display.

## Discover assets

All list methods use `start` and `end` pagination indexes.

```ts
const allAssets = await bond.getList({ start: 0, end: 100 })

const byCreator = await bond.getTokensByCreator({
  creator: '0x1111111111111111111111111111111111111111',
  start: 0,
  end: 100,
})

const byReserve = await bond.getTokensByReserveToken({
  reserveToken: '0x4200000000000000000000000000000000000006',
  start: 0,
  end: 100,
})
```

`getList()` returns Bond information records. The creator and reserve filters return asset addresses. Resolve a filtered address with `token(address)` or `nft(address)` only after determining its implementation type.

## Creator royalties

```ts
const account = '0x1111111111111111111111111111111111111111'
const reserveToken = '0x4200000000000000000000000000000000000006'

const royaltyInfo = await bond.getRoyaltyInfo({
  wallet: account,
  reserveToken,
})
```

Claiming is a write and requires a configured wallet:

```ts
await bond.claimRoyalties({
  reserveToken,
  onSuccess: (receipt) => console.log(receipt.transactionHash),
  onError: (error) => console.error(error),
})
```

## Safety

- Confirm the selected chain and reserve-token address.
- Treat pagination bounds as indexes, not counts.
- Read royalty state before asking the wallet to sign a claim.
- Do not assume returned asset addresses are ERC-20; Mint Club also supports ERC-1155 assets.
