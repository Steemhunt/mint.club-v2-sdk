# Debugging

Start with the earliest boundary in the request. Do not increase slippage or submit another approval until the network, asset, amount, and current quote are confirmed.

## 1. Confirm the network

```ts
import { supportedChainsMap } from '@mint.club/v2-sdk'

console.log(Object.keys(supportedChainsMap))
console.log(supportedChainsMap.base) // 8453
```

The wallet client's chain must match the SDK network used for a write.

## 2. Confirm the asset and units

```ts
import { mintclub, wei } from '@mint.club/v2-sdk'

const token = mintclub.network('base').token('SIGNET')

if (!(await token.exists())) throw new Error('Asset not found on Base')

const decimals = await token.getDecimals()
const amount = wei(100, decimals)
const reserveToken = await token.getReserveTokenAddress()
```

Never assume an ERC-20 has 18 decimals. Mint Club ERC-1155 assets use whole units such as `1n`.

## 3. Inspect balances and the current quote

```ts
const account = '0x...' as const
const network = mintclub.network('base')

const nativeBalance = await network.getPublicClient().getBalance({ address: account })
const tokenBalance = await token.getBalanceOf(account)
const reserveBalance = await network.token(reserveToken).getBalanceOf(account)
const [reserveNeeded] = await token.getBuyEstimation(amount)
```

For a buy, compare the reserve-token balance and allowance with `reserveNeeded`. For a sell, confirm the asset balance and Bond approval.

## 4. Confirm wallet configuration

```ts
const walletClient = mintclub.network('base').getWalletClient()

if (!walletClient?.account) throw new Error('Wallet account is not configured')
if (walletClient.chain?.id !== supportedChainsMap.base) {
  throw new Error('Wallet is connected to the wrong chain')
}
```

Application integrations should call `mintclub.withWalletClient(walletClient)` with a viem client that includes both `account` and `chain`.

## 5. Capture the transaction lifecycle

```ts
const receipt = await token.buy({
  amount,
  slippage: 1,
  debug: (simulationArgs) => console.log('Simulation', simulationArgs),
  onAllowanceSignatureRequest: () => console.log('Approval signature requested'),
  onAllowanceSigned: () => console.log('Approval signed'),
  onSignatureRequest: () => console.log('Trade signature requested'),
  onSigned: (hash) => console.log('Trade signed', hash),
  onError: (error) => console.error('Trade failed', error),
})

if (!receipt) console.error('No transaction receipt was returned')
```

The `debug` callback receives the SDK's simulation parameters. Inspect the object that is actually provided; do not assume extra fields such as a gas estimate are present.

Some write helpers route failures through `onError` and return `undefined`. A surrounding `try/catch` alone may not observe those failures.

## Common causes

| Symptom | Check |
|---|---|
| `ChainNotSupportedError` | Use a key from `supportedChainsMap` or its numeric ID. |
| `WalletNotConnectedError` | Configure a wallet client with an account and chain. |
| Insufficient funds | Check native gas balance and the relevant ERC-20/ERC-1155 balance. |
| Allowance or approval failure | Check the exact spender and amount; ERC-1155 uses operator approval. |
| Slippage revert | Refresh the quote and review market movement before changing tolerance. |
| Optional helper failure | Validate its deployment address and reject the sentinel `0x`. |
| USD rate is `null` | No supported price path was available for that token and network. |

Never log private keys, seed phrases, API credentials, or signed raw transactions while debugging.
