# Lockup operations

Use `mintclub.network(chain).lockup`. The Locker is optional, so validate its deployment address before approving an asset or constructing a write.

## Resolve the Locker and approve an asset

```ts
import {
  getMintClubContractAddress,
  mintclub,
  supportedChainsMap,
  wei,
} from '@mint.club/v2-sdk'

const network = mintclub.network('base')
const locker = getMintClubContractAddress('LOCKER', supportedChainsMap.base)

if (locker === '0x') throw new Error('Lockups are unavailable on Base')

const token = network.token('MYTOKEN')
const amount = wei(10_000, await token.getDecimals())

await token.approve({ spender: locker, amount })
```

For an ERC-1155 asset, approve the Locker as an operator:

```ts
const nft = network.nft('MYNFT')
await nft.approve({ spender: locker, approved: true })
```

The lockup helper does not submit token approval automatically.

## Create a lockup

```ts
await network.lockup.createLockUp({
  token: token.getTokenAddress(),
  isERC20: true,
  amount,
  unlockTime: Math.floor(Date.now() / 1000) + 30 * 24 * 60 * 60,
  receiver: '0x1111111111111111111111111111111111111111',
  title: 'Team allocation',
  onSuccess: (receipt) => console.log(receipt.transactionHash),
  onError: (error) => console.error(error),
})
```

`unlockTime` is a Unix timestamp in seconds. For ERC-1155 lockups, set `isERC20: false` and use a whole-unit `bigint` amount.

## Read lockups

```ts
const lockup = network.lockup

const total = await lockup.getTotalLockUpCount()
const details = await lockup.getLockUpById(1)

const received = await lockup.getLockUpIdsByReceiver({
  receiver: '0x1111111111111111111111111111111111111111',
  start: 0,
  end: 100,
})

const forToken = await lockup.getLockUpIdsByToken({
  token: token.getTokenAddress(),
  start: 0,
  end: 100,
})
```

## Unlock

```ts
await lockup.unlock({
  lockUpId: 1,
  onSuccess: (receipt) => console.log(receipt.transactionHash),
  onError: (error) => console.error(error),
})
```

Unlock succeeds only after the recorded time and transfers the asset to the recorded receiver.

## Safety

- Confirm the chain, asset type, amount units, receiver, and unlock timestamp.
- Verify the exact Locker spender before approval and reject the sentinel `0x`.
- Query ERC-20 decimals; ERC-1155 amounts are whole units.
- Read the lockup before unlocking to confirm its receiver, time, and claimed state.
