# Token and NFT operations

Use `token()` for ERC-20 assets and `nft()` for ERC-1155 assets. Both accept a Mint Club symbol or a contract address.

```ts
import { mintclub } from '@mint.club/v2-sdk'

const token = mintclub.network('base').token('SIGNET')
const nft = mintclub.network('base').nft('MYNFT')
```

## Common reads

```ts
await token.exists()
await token.getDetail()
await token.getTokenBond()
await token.getSteps()
await token.getMaxSupply()
await token.getTotalSupply()
await token.getReserveTokenAddress()
await token.getPriceForNextMint()
await token.getUsdRate({ amount: 1 })
```

ERC-20-specific reads include `getDecimals()`, `getAllowance()`, `getName()`, and `getSymbol()`. ERC-1155-specific reads include `getMetadataUri()`, `getContractURI()`, `getBalanceOfBatch()`, and `getIsApprovedForAll()`.

## Amounts, estimates, and trades

```ts
import { wei } from '@mint.club/v2-sdk'

const decimals = await token.getDecimals()
const amount = wei(100, decimals)

const [reserveNeeded] = await token.getBuyEstimation(amount)
const [reserveRefund] = await token.getSellEstimation(amount)

await token.buy({ amount, slippage: 1 })
await token.sell({ amount, slippage: 1 })
```

ERC-20 trade amounts use the token's smallest unit. Mint Club ERC-1155 assets have zero decimals, so use whole units:

```ts
await nft.buy({ amount: 1n, slippage: 1 })
await nft.sell({ amount: 1n, slippage: 1 })
```

The SDK checks Bond approvals and may request a separate approval signature before the trade signature.

## Create an ERC-20 asset

```ts
await mintclub.network('base').token('MYTOKEN').create({
  name: 'My Token',
  reserveToken: {
    address: '0x4200000000000000000000000000000000000006',
    decimals: 18,
  },
  curveData: {
    curveType: 'EXPONENTIAL',
    stepCount: 100,
    maxSupply: 1_000_000,
    initialMintingPrice: 0.0001,
    finalMintingPrice: 0.1,
    creatorAllocation: 100,
  },
  buyRoyalty: 3,
  sellRoyalty: 3,
})
```

Royalty values use percentage points: `3` means 3%. The SDK default `0.03` means 0.03%.

Pass `stepData` instead of `curveData` for explicit cumulative ranges:

```ts
await mintclub.network('base').token('STEP').create({
  name: 'Step Token',
  reserveToken: { address: '0x...', decimals: 18 },
  stepData: [
    { rangeTo: 100, price: 0 },
    { rangeTo: 1_000, price: 0.01 },
    { rangeTo: 10_000, price: 0.1 },
  ],
})
```

Pass exactly one of `curveData` and `stepData`.

## Create an ERC-1155 asset

ERC-1155 creation also requires `metadataUrl`:

```ts
await mintclub.network('base').nft('MYNFT').create({
  name: 'My Collection',
  metadataUrl: 'ipfs://...',
  reserveToken: { address: '0x...', decimals: 18 },
  stepData: [{ rangeTo: 1_000, price: 0.01 }],
})
```

## Approve and transfer

```ts
await token.approve({ spender: '0x...', amount })
await nft.approve({ spender: '0x...', approved: true })

await token.transfer({ recipient: '0x...', amount })
await nft.transfer({ recipient: '0x...', amount: 1n })
```

Confirm the exact spender before approving. ERC-1155 approval grants operator access to the collection.

## Wrapped-native Zap boundary

`buyWithZap()` and `sellWithZap()` use the SDK's legacy wrapped-native Zap contract. They work only when the asset's reserve is the selected network's wrapped native token and are not general any-token routing methods.

Use the separate Mint Club CLI/MCP tooling for routed ZapV2 flows through Universal Router.

## Safety

- Confirm the chain, asset implementation, address, reserve token, decimals, recipient, and slippage.
- Query estimates immediately before writes.
- Treat approval and trade signatures as separate actions.
- Some helpers report failures through `onError` and return `undefined`; require a receipt or `onSuccess` before reporting success.
