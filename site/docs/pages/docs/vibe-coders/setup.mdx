# Environment setup

## Install

Install the SDK alone for reads:

```bash
npm install @mint.club/v2-sdk
```

Install viem directly when application code creates clients or accounts:

```bash
npm install @mint.club/v2-sdk viem
```

## Read-only use

No wallet is required. The SDK creates a ranked fallback transport for each supported network.

```ts
import { mintclub } from '@mint.club/v2-sdk'

const details = await mintclub.network('base').token('SIGNET').getDetail()
```

Use [`withPublicClient()`](/docs/sdk/network/withPublicClient) when an application needs its own RPC provider.

## Browser wallet

Use a wallet client from wagmi or construct one with viem. It must include both an account and a chain.

```ts
import { mintclub, wei } from '@mint.club/v2-sdk'
import { createWalletClient, custom } from 'viem'
import { base } from 'viem/chains'

const [account] = await window.ethereum.request({
  method: 'eth_requestAccounts',
})

const walletClient = createWalletClient({
  account,
  chain: base,
  transport: custom(window.ethereum),
})

mintclub.withWalletClient(walletClient)

const token = mintclub.network('base').token('SIGNET')
const amount = wei(100, await token.getDecimals())

await token.buy({ amount, slippage: 1 })
```

For React, configure the SDK after `useWalletClient()` returns a client. Do not configure it during server rendering.

## Trusted server

Prefer constructing a viem wallet client so the account, chain, and transport are explicit:

```ts
import { mintclub } from '@mint.club/v2-sdk'
import { createWalletClient, http } from 'viem'
import { privateKeyToAccount } from 'viem/accounts'
import { base } from 'viem/chains'

const account = privateKeyToAccount(process.env.PRIVATE_KEY as `0x${string}`)
const walletClient = createWalletClient({
  account,
  chain: base,
  transport: http(process.env.BASE_RPC_URL),
})

mintclub.withWalletClient(walletClient)
```

The network-scoped `withPrivateKey()` helper is also available for trusted scripts:

```ts
const baseClient = mintclub
  .network('base')
  .withPrivateKey(process.env.PRIVATE_KEY as `0x${string}`)
```

Never place private keys in frontend bundles, prompts, logs, command arguments, or source control.

## Validate configuration

```ts
import { supportedChainsMap } from '@mint.club/v2-sdk'

const network = mintclub.network('base')
const walletClient = network.getWalletClient()

console.log('Supported chain ID:', supportedChainsMap.base)
console.log('Wallet account:', walletClient?.account?.address)
console.log('Wallet chain ID:', walletClient?.chain?.id)
```

The wallet chain must match the network used for a write. See [Debugging](/docs/vibe-coders/debugging) for quote, balance, allowance, and callback checks.
