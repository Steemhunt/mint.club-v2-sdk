# Airdrop operations

Use an asset helper to create an airdrop. Use the network-scoped `airdrop` helper for reads, claims, and cancellation.

## Create an airdrop

```ts
import { mintclub } from '@mint.club/v2-sdk'

const wallets = [
  '0x1111111111111111111111111111111111111111',
  '0x2222222222222222222222222222222222222222',
] as const

await mintclub.network('base').token('MYTOKEN').createAirdrop({
  title: 'Community airdrop',
  wallets: [...wallets],
  amountPerClaim: 100,
  startTime: new Date(),
  endTime: new Date(Date.now() + 7 * 24 * 60 * 60 * 1000),
  filebaseApiKey: process.env.FILEBASE_API_KEY as string,
})
```

For ERC-20 assets, `amountPerClaim` is a human-readable number and the SDK queries token decimals before conversion. For ERC-1155 assets, it is a whole-unit number.

The helper validates addresses, builds the Merkle root, uploads the allowlist, approves the Merkle distributor when required, and creates the distribution. A wallet may request both an approval signature and a distribution signature.

## Read state and eligibility

```ts
const airdrop = mintclub.network('base').airdrop
const airdropId = 1
const account = '0x1111111111111111111111111111111111111111'

const details = await airdrop.getAirdropById(airdropId)
const claimed = await airdrop.getAmountClaimed(airdropId)
const remaining = await airdrop.getAmountLeft(airdropId)
const hasClaimed = await airdrop.getIsClaimed(airdropId, account)
const isEligible = await airdrop.getIsWhitelisted(airdropId, account)

const owned = await airdrop.getAirdropIdsByOwner({ owner: account })
const forToken = await airdrop.getAirdropIdsByToken({ token: details.token })
```

`getIsWhitelisted()` fetches the stored allowlist and derives the Merkle proof when the distribution is private.

## Claim or cancel

```ts
await airdrop.claimAirdrop({
  airdropId,
  onSuccess: (receipt) => console.log(receipt.transactionHash),
  onError: (error) => console.error(error),
})

await airdrop.cancelAirdrop({ airdropId })
```

Only the distribution owner can cancel an eligible distribution. Contract time and claim-state rules still apply.

## Safety

- Confirm the chain, token, recipient list, amount per claim, and time window.
- Use valid EVM addresses and a future `endTime`.
- Keep Filebase credentials out of frontend bundles, logs, and prompts.
- Do not manually pass a Merkle proof to `claimAirdrop()`; the helper resolves it.
- Treat approval and creation as separate wallet-signing steps.
