# Staking operations

Use `mintclub.network(chain).stake`. Staking is an optional contract feature and is not deployed on every SDK-supported network.

## Validate the deployment and limits

```ts
import {
  getMintClubContractAddress,
  mintclub,
  supportedChainsMap,
  wei,
} from '@mint.club/v2-sdk'

const network = mintclub.network('base')
const stake = network.stake
const stakeAddress = getMintClubContractAddress('STAKE', supportedChainsMap.base)

if (stakeAddress === '0x') throw new Error('Staking is unavailable on Base')

const [creationFee, claimFee, minDuration, maxDuration] = await Promise.all([
  stake.getCreationFee(),
  stake.getClaimFee(),
  stake.getMinRewardDuration(),
  stake.getMaxRewardDuration(),
])
```

Fees and duration limits are live contract values. Do not hard-code them.

## Create a pool

Reward tokens are ERC-20 assets. Approve the Stake contract before creating a pool.

```ts
const rewardToken = network.token('REWARD')
const rewardAmount = wei(1_000, await rewardToken.getDecimals())

await rewardToken.approve({ spender: stakeAddress, amount: rewardAmount })

await stake.createPool({
  stakingToken: '0x1111111111111111111111111111111111111111',
  isStakingTokenERC20: true,
  rewardToken: rewardToken.getTokenAddress(),
  rewardAmount,
  rewardStartsAt: Math.floor(Date.now() / 1000) + 3600,
  rewardDuration: 7 * 24 * 60 * 60,
  value: creationFee,
  onSuccess: (receipt) => console.log(receipt.transactionHash),
})
```

The staking asset may be ERC-20 or ERC-1155. Set `isStakingTokenERC20` to match the contract.

## Stake and unstake

Approve the Stake contract to spend the staking asset before calling `stake()`.

```ts
const stakingToken = network.token('STAKE-TOKEN')
const amount = wei(100, await stakingToken.getDecimals())

await stakingToken.approve({ spender: stakeAddress, amount })
await stake.stake({ poolId: 1, amount })
await stake.unstake({ poolId: 1, amount })
```

For ERC-1155 staking assets, use operator approval and whole-unit amounts:

```ts
await network.nft('STAKE-NFT').approve({
  spender: stakeAddress,
  approved: true,
})

await stake.stake({ poolId: 1, amount: 1n })
```

## Read a position and claim

```ts
const account = '0x1111111111111111111111111111111111111111'

const pool = await stake.getPool({ poolId: 1 })
const position = await stake.getUserPoolStake({ user: account, poolId: 1 })
const claimable = await stake.getClaimableReward({ staker: account, poolId: 1 })

await stake.claim({
  poolId: 1,
  value: claimFee,
  onSuccess: (receipt) => console.log(receipt.transactionHash),
})
```

Other reads include:

```ts
await stake.getPoolCount()
await stake.getPools({ start: 0, end: 100 })
await stake.getPoolsByCreator({ creator: account, start: 0, end: 100 })
await stake.getClaimableRewardBulk({
  poolIdFrom: 0,
  poolIdTo: 100,
  staker: account,
})
```

Pool creators can call `cancelPool()`. Stakers can call `emergencyUnstake()` when appropriate; review the contract behavior before using either path.

## Administrative methods

The contract owner can call:

```ts
await stake.updateCreationFee({ creationFee: 1n })
await stake.updateClaimFee({ claimFee: 1n })
await stake.updateProtocolBeneficiary({ protocolBeneficiary: account })
await stake.transferOwnership({ newOwner: account })
```

## Safety

- Confirm the pool ID, token implementations, decimals, reward start, duration, and live fees.
- Reject an unavailable Stake address (`0x`).
- Treat token approvals and pool actions as separate transactions.
- Use `user` for `getUserPoolStake()` and `staker` for reward queries.
- Do not call owner-only methods from an unprivileged account.
