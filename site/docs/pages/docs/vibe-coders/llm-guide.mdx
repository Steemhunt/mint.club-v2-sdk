# LLM implementation guide

Use this page when generating code with `@mint.club/v2-sdk`. The repository also ships a plain-text [`llms-full.txt`](https://github.com/Steemhunt/mint.club-v2-sdk/blob/main/llms-full.txt) reference.

## Required reasoning order

1. Select a supported network key or chain ID.
2. Determine whether the asset is ERC-20 (`token`) or ERC-1155 (`nft`).
3. Resolve the symbol or address and confirm the asset exists.
4. Read decimals, reserve token, balances, and the current estimate.
5. Configure a wallet only when a write is required.
6. Show the user the recipient, amount, slippage, approvals, and signatures.

## Never guess units

```ts
import { mintclub, wei } from '@mint.club/v2-sdk'

const token = mintclub.network('base').token('SIGNET')
const decimals = await token.getDecimals()
const amount = wei(100, decimals)
const [reserveNeeded] = await token.getBuyEstimation(amount)
```

- ERC-20 amounts use the token's decimals.
- Mint Club ERC-1155 amounts are whole units.
- Slippage and royalties use percentage points: `1` means 1%.
- A reserve address or decimal count must come from onchain data or verified metadata.

## Reads

Reads do not need a wallet and use fallback RPC transports.

```ts
const asset = mintclub.network('base').token('SIGNET')

const [details, steps, usd] = await Promise.all([
  asset.getDetail(),
  asset.getSteps(),
  asset.getUsdRate({ amount: 1 }),
])
```

## Writes

Provide a viem wallet client with an account and chain:

```ts
mintclub.withWalletClient(walletClient)

await mintclub.network('base').token('SIGNET').buy({
  amount,
  slippage: 1,
  onAllowanceSignatureRequest: () => console.log('Approval requested'),
  onSignatureRequest: () => console.log('Trade requested'),
  onSuccess: (receipt) => console.log(receipt.transactionHash),
  onError: (error) => console.error(error),
})
```

For trusted Node.js processes, `withPrivateKey()` is available after `network()`:

```ts
await mintclub
  .network('base')
  .withPrivateKey(process.env.PRIVATE_KEY as `0x${string}`)
  .token('SIGNET')
  .sell({ amount, slippage: 1 })
```

Never put a private key in browser code, a prompt, logs, source control, or command-line arguments.

## Network keys

Mainnets: `ethereum`, `base`, `blast`, `optimism`, `arbitrum`, `polygon`, `bnbchain`, `avalanche`, `kaia`, `cyber`, `degen`, `ham`, `shibarium`, `unichain`, `zora`, `apechain`, `hashkey`, `robinhood`, `over`.

Testnets: `sepolia`, `basesepolia`, `blastsepolia`, `avalanchefuji`, `cybertestnet`, `puppynet`.

Optional contracts and individual price providers are not deployed on every network. Validate addresses returned by `getMintClubContractAddress()` and reject the sentinel `0x`.

## Zap boundary

The SDK's `buyWithZap()` and `sellWithZap()` methods are wrapped-native ZapV1 helpers. They require the asset reserve to be the network's wrapped native token and are not general any-token routing methods.

Use the separate Mint Club CLI/MCP tooling for routed ZapV2 flows through Universal Router.

## Error handling

Some SDK write helpers pass errors to `onError` and return `undefined`. Treat a write as successful only when a receipt is returned or `onSuccess` runs.

Do not suppress approval prompts, wallet rejections, RPC failures, contract reverts, or quote failures. Return them to the caller with the attempted operation and parameters, excluding secrets.
