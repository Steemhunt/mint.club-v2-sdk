# Contracts

Mint Club V2 is implemented as a set of onchain contracts for bonding curves, ERC-20 and ERC-1155 assets, lockups, airdrops, staking, and optional Zap flows.

## Source and security material

- [Contract source and deployment registry](https://github.com/Steemhunt/mint.club-v2-contract)
- [Protocol documentation](https://docs.mint.club)
- [Community audit](https://docs.mint.club/audit/com_audit)
- [CertiK audit](https://docs.mint.club/audit/report)

The contract repository is the canonical source for deployment tables, verification links, test coverage, and gas reports. Those values change independently of the SDK, so they are not duplicated here.

## Resolve an address from the SDK

The SDK bundles the addresses required by its contract helpers. Use `getMintClubContractAddress()` when an integration needs the address for its selected chain.

```ts
import { getMintClubContractAddress, supportedChainsMap } from '@mint.club/v2-sdk'

const chainId = supportedChainsMap.base

const bond = getMintClubContractAddress('BOND', chainId)
const tokenFactory = getMintClubContractAddress('ERC20', chainId)
const multiTokenFactory = getMintClubContractAddress('ERC1155', chainId)
```

Supported contract keys are:

```ts
type ContractName =
  | 'BOND'
  | 'ERC20'
  | 'ERC1155'
  | 'LOCKER'
  | 'MERKLE'
  | 'ONEINCH'
  | 'STAKE'
  | 'ZAP'
```

:::warning
Not every optional contract is deployed on every SDK-supported network. Some unavailable entries use the sentinel value `0x`; validate the result before constructing a transaction.
:::

## Low-level contract access

Typed contract wrappers are exported for advanced integrations:

```ts
import { bondContract, erc20Contract, erc1155Contract } from '@mint.club/v2-sdk'

const creationFee = await bondContract.network('base').read({
  functionName: 'creationFee',
})
```

For common operations, prefer the higher-level `token()`, `nft()`, `airdrop`, `lockup`, and `stake` helpers documented in this site.
