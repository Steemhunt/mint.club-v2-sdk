# Mint Club V2 SDK — airdrop reference

Package: `@mint.club/v2-sdk`

Use `token(symbolOrAddress).createAirdrop()` or `nft(symbolOrAddress).createAirdrop()` for the high-level creation flow. Use `network(chain).airdrop` for reads, claims, and cancellation.

All writes require a configured wallet. Never expose the Filebase API key in frontend code or logs.

## Create

```ts
import { mintclub } from '@mint.club/v2-sdk'

const wallets = [
  '0x1111111111111111111111111111111111111111',
  '0x2222222222222222222222222222222222222222',
] as const

await mintclub.network('base').token('MYTOKEN').createAirdrop({
  title: 'Community airdrop',
  wallets: [...wallets],
  amountPerClaim: 100,
  startTime: new Date(),
  endTime: new Date(Date.now() + 7 * 24 * 60 * 60 * 1000),
  filebaseApiKey: process.env.FILEBASE_API_KEY as string,
})
```

For ERC-20 assets, `amountPerClaim` is a human-readable number and the SDK queries decimals before conversion. For ERC-1155 assets, it is a whole-unit number.

The high-level helper validates wallet addresses, creates the Merkle root, uploads the allowlist, handles the Merkle contract approval, and creates the onchain distribution.

## Read eligibility and state

```ts
const airdrop = mintclub.network('base').airdrop
const airdropId = 1
const account = '0x1111111111111111111111111111111111111111'

await airdrop.getAirdropById(airdropId)
await airdrop.getAmountClaimed(airdropId)
await airdrop.getAmountLeft(airdropId)
await airdrop.getIsClaimed(airdropId, account)
await airdrop.getIsWhitelisted(airdropId, account)
await airdrop.getAirdropIdsByOwner({ owner: account })
await airdrop.getAirdropIdsByToken({ token: '0x...' })
```

## Claim or cancel

```ts
await airdrop.claimAirdrop({
  airdropId,
  onSuccess: (receipt) => console.log(receipt.transactionHash),
  onError: (error) => console.error(error),
})

await airdrop.cancelAirdrop({ airdropId })
```

The claim helper fetches the stored allowlist and constructs the proof automatically. Only the distribution owner can cancel an eligible distribution.

## Safety

- Confirm the token, recipient list, amount per claim, start time, and end time.
- Use valid EVM addresses and a future `endTime`.
- Do not pass a manually constructed proof to `claimAirdrop()`.
- Treat Filebase and wallet credentials as secrets.
