# Mint Club V2 SDK — trade reference

Package: `@mint.club/v2-sdk`

Use `mintclub.network(chain).token(symbolOrAddress)` for ERC-20 assets and `.nft(symbolOrAddress)` for ERC-1155 assets.

## Wallet requirement

Reads do not require a wallet. Writes require a viem wallet client with an account and chain:

```ts
mintclub.withWalletClient(walletClient)
```

For a trusted server, `withPrivateKey()` is network-scoped:

```ts
const baseClient = mintclub
  .network('base')
  .withPrivateKey(process.env.PRIVATE_KEY as `0x${string}`)
```

Never expose or request a private key in frontend or agent code.

## Units

- Trade `amount` is a `bigint` in the asset's smallest unit.
- Query ERC-20 decimals; do not assume 18.
- Mint Club ERC-1155 assets use whole units such as `1n`.
- `slippage` uses percentage points: `1` means 1%, `0.5` means 0.5%.

## Quote and trade an ERC-20

```ts
import { mintclub, wei } from '@mint.club/v2-sdk'

const token = mintclub.network('base').token('SIGNET')
const decimals = await token.getDecimals()
const amount = wei(100, decimals)

const [reserveNeeded] = await token.getBuyEstimation(amount)
const [reserveRefund] = await token.getSellEstimation(amount)

await token.buy({
  amount,
  slippage: 1,
  recipient: '0x...',
  onSuccess: (receipt) => console.log(receipt.transactionHash),
  onError: (error) => console.error(error),
})

await token.sell({ amount, slippage: 1 })
```

`buy()` mints the exact asset amount and derives a maximum reserve input. `sell()` burns the exact asset amount and derives a minimum reserve output.

The SDK checks the relevant allowance. If needed, the same trade call submits and waits for an approval before continuing, so the wallet may request two signatures.

## Trade an ERC-1155 asset

```ts
const nft = mintclub.network('base').nft('MYNFT')

await nft.buy({ amount: 1n, slippage: 1 })
await nft.sell({ amount: 1n, slippage: 1 })
```

ERC-1155 sells use operator approval rather than an ERC-20 allowance.

## Wrapped-native Zap boundary

`buyWithZap()` and `sellWithZap()` use the SDK's legacy wrapped-native Zap contract. They work only when the asset reserve is the selected network's wrapped native token.

```ts
await token.buyWithZap({ amount, slippage: 1 })
await token.sellWithZap({ amount, slippage: 1 })
```

These methods are not general any-token routing. Use the separate Mint Club CLI/MCP tooling for routed ZapV2 flows through Universal Router.

## Failure handling

Write methods accept `onAllowanceSignatureRequest`, `onAllowanceSigned`, `onAllowanceSuccess`, `onSignatureRequest`, `onSigned`, `onSuccess`, and `onError` callbacks.

Some helpers report failures through `onError` and return `undefined`. Treat a write as successful only when a receipt is returned or `onSuccess` runs.
